# 仓库管理系统迁移方案

## 一、概述

将袋鼠君小程序（PHP/ThinkPHP 后端）的仓库管理功能迁移到 NestJS 后端。

### PHP 参考源码
- `ref/php-api/Stores.php` — 仓库管理控制器（767行，16个API）
- 相关模型: `OrderModel`, `ShipOrders`, `Configs`, `Printer`
- 相关库: `MnpAlertLogic`, `Wecom`

### NestJS 目标路径
- `~/workspace/kangaroo-japan-backend/src/warehouse/` — 仓库管理模块

## 四、代码架构

### 4.1 模块结构

```
src/warehouse/
├── warehouse.module.ts          # 模块定义
├── warehouse.controller.ts       # 控制器（路由映射）
├── warehouse.service.ts          # 业务逻辑
├── entities/
│   ├── shipment-order.entity.ts  # 出库单实体
│   └── order-photo.entity.ts     # 订单照片实体
├── dto/
│   ├── orders-query.dto.ts       # 待入库订单查询
│   ├── instore.dto.ts            # 入库请求
│   ├── cancelstore.dto.ts        # 取消入库请求
│   ├── ships-query.dto.ts        # 发货单查询
│   ├── checkship.dto.ts          # 审核发货请求
│   ├── confirm-ship.dto.ts       # 盘货确认请求
│   ├── doship.dto.ts             # 执行出库请求
│   └── add-photos.dto.ts         # 提交照片请求
└── interfaces/
    └── warehouse.interface.ts    # 接口定义
```

### 4.2 关键业务流程

#### 入库流程
1. `orders()` → 查询 status=2(待入库) 且 is_pay=1 的订单列表
2. `instore(id, weight, area)` → 将订单 status=2→3(已入库), 记录 weight/store_time/store_area
3. `cancelstore(id)` → 将订单 status=3→2(恢复待入库), 清空 weight/store_time

#### 出库发货流程
1. `ships()` → 查询出库单列表（关联用户、物流方式）
2. `checkship(id)` → 查看出库单详情（关联订单、用户等级、汇率）
3. `confirm(id, weight, post_fee, pack_fee, amount)` → 审核确认，计算金额
4. `doship(id, post_code)` → 执行发货，更新出库单和关联订单状态

## 六、安全与认证

- 所有 API 通过 `JwtAuthGuard` 保护
- 管理员权限通过 `RolesGuard` 控制（需要 `UserRole.ADMIN`）
- 验证通过 `class-validator` DTO 实现
