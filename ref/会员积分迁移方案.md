# 会员等级 + 积分商城模块迁移方案

**目标**：将 PHP 端会员等级和积分商城模块迁移到现有 NestJS 后端

**背景分析**：
- PHP 数据库使用 MySQL（表名前缀 `st_`），NestJS 使用 PostgreSQL
- NestJS 已有用户模块（`src/users/`）、支付模块（`src/payments/`）、JWT 认证
- 现有 `User` 实体已有 `depositBalance` 等字段，但缺少 `score`、`level`、`level_end_time`
- 支付系统已封装 Stripe/Pingxx，VIP 购买应复用该流程

### 2. User 实体扩展字段（直接在 `src/users/user.entity.ts` 追加）

```typescript
@Column({ type: 'int', default: 1 })
level: number;               // 当前等级ID（默认1=普通）

@Column({ name: 'level_end_time', type: 'timestamp', nullable: true })
levelEndTime: Date;          // 会员到期时间

@Column({ type: 'int', default: 0 })
score: number;               // 当前积分

@Column({ name: 'score_total', type: 'int', default: 0 })
scoreTotal: number;          // 累计获得积分
```

### 3. API 设计

#### POST /api/v1/user/levels → 获取会员等级

- **认证**: JWT 必选
- **参数**: `{}`
- **逻辑**:
  1. 获取当前用户信息（level, levelEndTime, nickname, avatarUrl）
  2. 查询 `user_levels` 表全部等级，按 `level` 升序
  3. 计算每个等级的 `currentLevelStatus`: 0=低于、1=当前、-1=高于
  4. 计算差额 `offsetAmount`：仅当用户有有效会员且目标等级更高时
  5. 返回等级列表、用户信息、购买时长选项、规则说明

#### POST /api/v1/user/vipbuy → 购买VIP会员

- **认证**: JWT 必选
- **参数**: `{"level": 2, "month": 0}`
- **逻辑**:
  1. 校验 level in [2,3,4]，month索引有效
  2. 查询等级信息
  3. 计算金额（基础价+升级补差价）
  4. 生成 out_trade_no
  5. 创建 VipOrder 记录

#### 支付回调处理
在 VipOrdersService 添加支付成功回调：标记已支付，更新用户 level 和 level_end_time

## 二、积分商城模块 (ScoreShop)

### 1. 实体设计（3张新表）

#### 1.1 `coupons` —— 优惠券/积分商品定义表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `uuid` | 主键 |
| `name` | `varchar(100)` | 名称 |
| `icon` | `varchar(500)` | 图标URL |
| `type` | `varchar(20)` | 类型（rate=折扣, cash=代金券） |
| `order_type` | `varchar(50)` | 适用订单类型 |
| `data` | `decimal(10,2)` | 面值 |
| `condition` | `decimal(10,2)` | 满减条件 |
| `expire_days` | `int` | 有效期（天） |
| `stock` | `int` | 库存 |
| `number` | `int` | 已发放数量 |
| `score` | `int` | 兑换所需积分 |
| `canbuy` | `boolean` | 是否可兑换 |
| `act_type` | `varchar(30)` | 活动类型 |
| `act_extras` | `jsonb` | 额外配置 |
| `is_deleted` | `boolean` | 软删除 |

**Entity**: `src/score-shop/entities/coupon.entity.ts`

#### 1.2 `user_coupons` —— 用户已领取优惠券

| 字段 | 类型 |
|------|------|
| id, user_id, coupon_id, code, type, order_type, name, icon, condition, data, expire, source, used_at, order_id, is_used, created_at |

**Entity**: `src/score-shop/entities/user-coupon.entity.ts`

#### 1.3 `score_logs` —— 积分变动记录

| 字段 | 类型 |
|------|------|
| id, user_id, amount, type, remark, before_score, after_score, created_at |

**Entity**: `src/score-shop/entities/score-log.entity.ts`

### 2. API 设计

#### POST /api/v1/score/shop/goods → 积分商品列表
参数: {page, pageSize}
返回: {list: [...], total}

#### POST /api/v1/score/shop/buy → 积分兑换商品
参数: {id}
逻辑: 事务（扣积分→发券→减库存）

#### POST /api/v1/score/shop/getcoupon → 领取优惠券
参数: {id, type}
逻辑: 校验+频率限制+发券
